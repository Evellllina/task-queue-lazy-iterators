# Очередь задач: итераторы и генераторы
## Цель:
Научиться реализовывать пользовательские коллекции и ленивую обработку
задач
## Функциональность:
1) реализация протокола итерации;
2) поддержка повторного обхода очереди;
3) реализация ленивых фильтров
4) корректная работа с большими объёмами задач;

## Вывод:
В ходе выполнения лабораторной работы была реализована очередь задач TaskQueue с поддержкой:
1) протокола итерации через __iter__ и __next__ в отдельном классе итератора
2) повторного обхода каждый вызов __iter__ создаёт новый итератор, что позволяет многократно проходиться по очереди
3) ленивых фильтров методы filter_by_status, filter_by_priority и filter используют генераторы (yield), что исключает избыточное хранение данных в памяти
4) корректной обработки StopIteration итератор штатно завершает обход при исчерпании задач
